# Every Code vs OpenAI Codex 增强对比

Every Code 是 OpenAI Codex 的社区驱动 fork，专注于开发者实际体验。以下是主要增强：

## 核心增强功能

### 1. 🚀 Auto Drive 编排系统

**上游 Codex：** 基础的对话式编码

**Every Code 增强：**
- **多代理自动化** - 多个 CLI 代理协同工作
- **自愈能力** - 失败后自动调整策略并重试
- **完整任务交付** - 不只是生成代码，而是完成整个任务
- **解耦架构** - Auto Drive 和 Auto Review 分离，后台审查不阻塞主流程
- **有界状态管理** - 硬性限制内存使用，防止长时间运行导致内存泄漏

**新模型支持：**
- `gpt-5.3-codex` - 规划和问题解决
- `gpt-5.3-codex-spark` - 快速编码/修复循环
- `medium | high | xhigh` 推理级别控制

### 2. 🌐 浏览器深度集成

**上游 Codex：** 无浏览器支持

**Every Code 增强：**
- **CDP 支持** - Chrome DevTools Protocol 完整支持
- **无头浏览** - 可在后台运行浏览器
- **内联截图** - 直接在对话中查看页面状态
- **浏览器自动化工具** - 点击、填写表单、导航等
- **视觉验证** - 自动验证 Web 应用功能

### 3. 🤖 多代理命令系统

**上游 Codex：** 单一对话模式

**Every Code 增强：**
- **`/plan`** - 规划复杂任务，多个代理分析项目结构
- **`/code`** - 执行编码任务，多个代理协同实现
- **`/solve`** - 并行运行多种方案，比较结果
- **代理编排** - 自动协调不同 AI 代理（Claude、Gemini、Qwen 等）

### 4. 🔍 Auto Review 后台审查

**上游 Codex：** 无自动审查

**Every Code 增强：**
- **后台 Ghost Commit** - 在独立 worktree 运行审查
- **非阻塞** - 审查不阻塞主线程，用户可以继续工作
- **智能修复** - 使用 `codex-5.1-mini-high` 生成修复建议
- **并行执行** - 与 Auto Drive 任务并行运行
- **质量优先** - 从"模型能否写这个文件"转向"我们是否验证它有效"

### 5. 🌉 Code Bridge 桥接器

**上游 Codex：** 无应用集成

**Every Code 增强：**
- **Sentry 风格本地桥接** - 流式传输错误、控制台、截图
- **运行应用控制** - 直接从 Code 控制运行中的应用
- **MCP 服务器** - 扩展工具协议支持
- **一键安装** - `https://github.com/just-every/code-bridge`

### 6. 🎨 主题系统

**上游 Codex：** 固定主题

**Every Code 增强：**
- **多预设主题** - 浅色/深色多套预设
- **实时预览** - `/themes` 命令实时预览
- **自定义配色** - 可自定义主色、边框、光标等
- **可访问性优化** - 专为不同视觉需求设计

### 7. 🧩 统一设置中心

**上游 Codex：** 分散的配置

**Every Code 增强：**
- **`/settings` 覆盖层** - 统一管理所有设置
- **限制控制** - Token 限制、时间限制等
- **审批策略** - 细粒度的命令审批控制
- **提供商配置** - 统一的 AI 提供商管理

### 8. 🔌 MCP 扩展协议

**上游 Codex：** 基础工具调用

**Every Code 增强：**
- **文件系统扩展** - 更强大的文件操作
- **数据库集成** - 直接查询数据库
- **API 连接** - 调用外部 API
- **自定义工具** - 轻松添加自己的工具

### 9. 🌏 多模型提供商支持

**上游 Codex：** 主要支持 OpenAI

**Every Code 增强：**
- **DeepSeek** - `deepseek-chat`, `deepseek-reasoner`, `deepseek-coder`
- **智谱 GLM** - `glm-5`, `glm-4.7`, `glm-4.7-flashx`
- **Kimi (Moonshot)** - `kimi-k2.5`, `moonshot-v1-128k`
- **模型族抽象** - 统一接口支持多种提供商
- **自定义提供商** - 轻松添加任何 OpenAI 兼容的 API

### 10. 🔒 增强的安全模式

**上游 Codex：** 基础沙箱

**Every Code 增强：**
- **Linux Landlock** - 内核级沙箱隔离
- **细粒度权限** - 读写权限精确控制
- **命令审批** - 危险命令需要用户确认
- **环境变量过滤** - 自动过滤敏感信息

## 技术架构改进

### 稳定性增强
- **有界队列** - Auto Drive 对话/更新队列有界限制
- **确定性停止** - TUI 清理生命周期有确定性停止控制
- **压缩归档** - 终端代理压缩和归档，减少负载
- **压力测试** - 覆盖高代理流失和并发场景

### 性能优化
- **更快的历史记录** - 基于卡片的活动流
- **更可靠的流式传输** - 优化的流式处理
- **改进的 `/resume`** - 更可靠的会话恢复
- **改进的 `/undo`** - 更好的撤销功能

## 用户体验提升

### 交互改进
- **即时 Esc 响应** - 立即返回控制，输入无延迟
- **后台审查可见** - 审查完成后元数据仍可查询
- **并行处理** - 用户可以继续工作，后台完成审查

### 可定制性
- **代理模型别名** - 支持兼容性别名升级
- **主题自定义** - 深度定制 UI 外观
- **提供商切换** - 轻松切换不同 AI 提供商

## 总结

| 特性 | OpenAI Codex | Every Code |
|------|--------------|------------|
| Auto Drive | 基础 | 多代理编排 + 自愈 |
| 浏览器集成 | ❌ | ✅ CDP + 截图 |
| 多代理命令 | ❌ | ✅ /plan /code /solve |
| Auto Review | ❌ | ✅ 后台并行审查 |
| Code Bridge | ❌ | ✅ 应用桥接 |
| 主题系统 | 固定 | ✅ 多预设 + 自定义 |
| MCP 扩展 | 基础 | ✅ 丰富扩展 |
| 多模型支持 | OpenAI | ✅ 10+ 提供商 |
| 中文模型 | ❌ | ✅ DeepSeek/智谱/Kimi |

**核心差异：**
- OpenAI Codex - 官方版本，稳定但保守
- Every Code - 社区驱动，实验性强，功能更丰富

Every Code 在保持与上游兼容的同时，大幅增强了实际开发场景的可用性。
